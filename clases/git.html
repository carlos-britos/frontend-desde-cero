Git vs GitHub

Git

Herramienta en tu compu

No necesita internet

Maneja tu historial local

GitHub

Plataforma web

Necesita internet

Aloja tu repo en la nube

Git es como Word. GitHub es como Google Drive. Uno crea los documentos, el otro los guarda en la nube.

Git fue creado en 2005 por Linus Torvalds (el mismo que creo Linux). Hoy lo usa el 95%+ de los equipos de desarrollo del mundo.
Seccion 2

El repositorio — tu proyecto con memoria

Como se crea un repositorio

Crear uno nuevo

git init

Le dice a Git: "Empeza a seguir los cambios de esta carpeta"

Bajarte uno que ya existe

git clone <url>

Te baja la carpeta completa con todo su historial

No toques NUNCA la carpeta .git. Es la memoria de Git. Si la borras, perdes todo el historial.

Seccion 3

Commits — las fotos de tu proyecto

Imaginate que estas renovando tu depto. Cada vez que terminas algo importante (pintaste una pared, pusiste un mueble, cambiaste el piso), sacas una foto y le pones una nota. Un commit es exactamente eso.

Working Directory
Tus archivos (donde editas)
git add →
Staging Area
La zona de preparacion
git commit →
Repository
El historial (commits)

Pensalo asi: estas en un super. git add es meter productos al carrito. git commit es pasar por caja. Lo que no pusiste en el carrito queda en la gondola esperando.

Buenos mensajes de commit

Mal

"cambios"
"arregle cosas"
"wip"
"asdfghj"

Bien

"Agrego formulario de contacto"
"Corrijo error en validacion de email"
"Agrego estilos al header"

El mensaje tiene que explicar QUE hiciste de forma que si lo lees en 3 meses, entiendas sin abrir el codigo.

Seccion 4

Ramas — universos paralelos

Comandos de ramas

git branch

Ver en que rama estas

git switch -c nombre

Crear rama y moverte

git switch main

Volver a main

Convenciones de nombres de ramas:

feature/nombre-de-lo-que-haces fix/que-estas-arreglando

Siempre en minusculas, sin espacios (usar guiones).

Seccion 5

El repositorio remoto y GitHub

Paso a paso para conectar con GitHub

1

Crear repo en GitHub (boton "New repository")

2

Conectar

git remote add origin <url>
3

Subir

git push -u origin main
4

Verificar en GitHub que aparecio tu codigo

Subir codigo a GitHub NO es lo mismo que publicar una pagina web. GitHub guarda el codigo, no lo ejecuta.

Seccion 6

Flujo de trabajo en equipo

↓
git pull
Bajo lo ultimo de main
⑂
git switch -c feature/mi-tarea
Creo mi espacio de trabajo
●
git add . && git commit -m "..."
Trabajo y voy guardando
↑
git push
Subo mi trabajo a GitHub
✓
Abrir Pull Request
Pido que revisen y aprueben

Un PR es como el control de calidad en una fabrica. Vos armas la pieza (tu rama). Antes de que entre a produccion (main), alguien la revisa. Si esta bien, se aprueba. Si hay algo para corregir, te lo devuelven con notas.

Seccion 7

Conceptos que vas a escuchar

Merge y conflictos

Cuando dos personas tocan la misma linea, Git te pide que elijas. No te asustes: es normal y tiene solucion.

.gitignore

node_modules/
.env
.DS_Store

Este archivo le dice a Git que cosas NO seguir. Cosas pesadas, secretos, archivos basura del sistema.

Preguntas frecuentes

Necesitas internet para usar Git?

No. Git funciona 100% local. Solo necesitas internet para push/pull.

GitHub es la unica opcion?

No. Existen GitLab, Bitbucket, etc. La herramienta (Git) es la misma.

Push y commit son lo mismo?

No. Commit guarda local. Push sube al remoto. Primero commit, despues push.
